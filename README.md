## What problem does Bus Scope solve?

USB debugging is often trapped between expensive hardware analyzers, raw USBPcap dumps, kernel logs, and firmware printf output. Bus Scope gives engineers a local desktop view of the bus so descriptor mismatches, endpoint stalls, HID reports, CDC traffic, and mass-storage evidence can be reviewed in one place.

## The result

Use Bus Scope when you need a concrete result, not another generic tool window:

- Windows USBPcap and Linux usbmon capture workflows.
- Device, configuration, interface, endpoint, HID, and class evidence.
- Endpoint, direction, transfer type, text, and hex filters.
- Packet detail with setup fields, raw bytes, decoded evidence, and diagnostics.

## Who it is for

- Firmware engineers validating enumeration, control transfers, endpoint behavior, and class-specific traffic.
- Driver and support teams that need a readable USB report before blaming host, cable, driver, or device firmware.
- Hardware teams that need a cheaper daily workflow before escalating to a dedicated hardware analyzer.

## Product capabilities

- Windows USBPcap and Linux usbmon capture workflows
- Device, configuration, interface, endpoint, HID, and class evidence
- Endpoint, direction, transfer type, text, and hex filters
- Packet detail with setup fields, raw bytes, decoded evidence, and diagnostics
- Save/open .bscope sessions for lab notes and regression cases
- JSON, HTML, text, and paid PDF-style report export
